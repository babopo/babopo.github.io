  • 在浏览器中，浏览器会在js和图片等文件解析执行后直接存入内存缓存中，那么当刷新页面时只需直接从内存缓存中读取(from memory cache)；而css文件则会存入硬盘文件中，所以每次渲染页面都需要从硬盘读取缓存(from disk cache)。

  • 对node中require的理解

    • http://www.ruanyifeng.com/blog/2015/05/require.html

    • node里每一个模块的全局global下都有一个属性module(module并不是global的属性，对每一个模块来说其作用域中都有一个module)都是一个构造函数Module的实例，注意这个module对象是在该文件被require时从require它的文件创建并传入这个文件被包装后的函数的

      • function Module(id, parent) {
  this.id = id;
  this.exports = {};
  this.parent = parent;
  this.filename = null;
 //模块的绝对路径 
  this.loaded = false;
  this.children = [];
  this.paths: [];
  //数组，包含模块的可能位置
}
var module = new Module(filename, parent);

    • require是Module实例的一个方法(和全局中的require不是同一个函数)，它会调用Module构造函数上的_load方法

      • 对于_load方法，遵循：计算绝对路径->缓存->内置模块->生成Module实例module存入缓存，执行实例的load方法从前面的路径中加载->输出module.exports

    • 模块的加载js文件的实质上就是，注入exports、require、module三个全局变量，然后执行模块的源码，然后将模块的 exports 变量的值输出

      • node在加载js文件时会对js文件内容进行头尾包装，形式如下

      • (function (exports, require, module, __filename, __dirname) {
    xxxx
})

      • 这几个参数属性并不存在于模块文件中，其中__filename和__dirname是查找文件路径的过程中分析得到后传入的，module是这个模块自身，exports是指向module.exports的引用。这也是为什么js文件中可以不声明直接使用这几个变量，而且应当禁止这几个变量作为他用。

      • 同时根据前面的内容，同一个js文件中module.require !== require，因为require作为形参传入了，实际上内容相同但是是不同的引用。

    • 最后总结一下流程：当require一个js文件时，先计算绝对路径，判断模块是否有缓存，是否为内置模块；然后用Module构造函数生成一个module对象存入缓存，这个对象上有exports, require, __dirname, __filename等属性，其中exports属性初始化为一个空对象，__dirname, __filename是计算出来的待引入模块的路径；将待引入模块文件读取后包装成一个函数，传入前面这些属性，然后立即执行这个函数(模块中将需要的内容存入了exports)，即将模块引入并存入了缓存。

    • web worker：通过Worker构造函数接收js文件开启，多线程解决方案，通过事件驱动
      • 缺陷：同源限制，不能访问dom结点(因为是单独的作用域)

  • 关于箭头函数的this，既然箭头函数的this是创建时绑定的，而bind能返回一个新的函数，那么能否用bind改变箭头函数的this

    • 通过babel中测试可以知道，我们在箭头函数中使用的this其实都被编译成了一个指向外部作用域的this的变量，实际上我们我们并没有使用真正的this，因此bind/apply/call的改变this指向功能都无效

  • 关于Vue监听数组

    • vue对数组的监听和普通的变量不同，因为数组经常要做遍历，所以将length等属性修改为getter，setter在调用时消耗过大，vue的做法是对数组原型上的方法修改来实现拦截，所以直接给数组的索引上赋值字面量/直接修改数组长度是不会被监听的。
    • 引申思考：在拦截ajax，动态jsonp等请求时，也可以在其构造函数上进行修改来达到拦截的目的

  • react中不推荐使用字符串ref的原因，推荐现在使用回调函数的形式，而且现在有React.createRef了

    • 回调函数形式会在传入ref属性的组件挂载，更新，卸载这几个生命周期中执行；且在卸载之后执行时callback接收到的参数为null；这么做的目的在于组件销毁时对变量的回收，且使用回调函数可以做更多的事情

  • 浏览器可以并行下载，那么为什么合并http请求可以提高性能：由于http/1.1的keep-alive，无论多少条请求都只有三次握手，但是一些复杂逻辑和服务求浪费的时间无法忽略

    • pipeling的原理是客户端可以并行发送多个请求，但是服务器的响应必须按次序返回
